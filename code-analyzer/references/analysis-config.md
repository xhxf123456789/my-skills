## 静态分析配置

### pylint 配置文件
可在项目根目录创建 `.pylintrc` 文件自定义规则：

```ini
[MESSAGES CONTROL]
# 禁用特定检查
disable=C0114,C0115,C0116,C0301

[FORMAT]
# 最大行长度
max-line-length=100

[BASIC]
# 变量命名规范
variable-rgx=[a-z_][a-z0-9_]{2,30}$
```

### 常用禁用规则
| 规则代码 | 说明 | 适用场景 |
|---------|------|---------|
| C0114 | 缺少模块文档字符串 | 快速原型开发 |
| C0115 | 缺少类文档字符串 | 快速原型开发 |
| C0116 | 缺少函数文档字符串 | 快速原型开发 |
| C0301 | 行过长 | 保持代码可读性 |
| W0611 | 未使用的导入 | 清理导入 |

### 分析规则优先级
1. **Error（错误）**：必须修复，会导致程序崩溃或异常
2. **Warning（警告）**：建议修复，可能导致潜在问题
3. **Refactor（重构）**：建议优化，提高代码质量
4. **Convention（约定）**：代码风格问题
5. **Info（信息）**：一般性提示

## 复杂度配置

### 复杂度阈值建议
| 复杂度范围 | 等级 | 建议操作 |
|-----------|------|---------|
| 1-5 | A（简单） | 无需重构 |
| 6-10 | B（中等） | 考虑拆分 |
| 11-20 | C（复杂） | 建议重构 |
| 21-30 | D（很复杂） | 强烈建议重构 |
| 31-40 | E（极复杂） | 必须重构 |
| 41+ | F（不可接受） | 必须立即重构 |

### 复杂度优化策略
1. **提取方法**：将复杂逻辑拆分为多个小方法
2. **使用多态**：替换复杂的条件语句
3. **使用策略模式**：减少分支逻辑
4. **简化条件表达式**：使用提前返回、合并条件

### 配置示例
```bash
# 设置最小复杂度阈值为 8
python scripts/complexity_analyzer.py --path ./myproject --min-complexity 8
```

## 重复检测配置

### 相似度阈值说明
| 阈值范围 | 检测精度 | 适用场景 |
|---------|---------|---------|
| 0.9-1.0 | 高精度 | 查找完全相同的代码块 |
| 0.8-0.9 | 平衡模式 | 推荐默认设置 |
| 0.7-0.8 | 宽松模式 | 查找相似逻辑 |

### 最小行数建议
| 项目规模 | 建议行数 | 说明 |
|---------|---------|------|
| 小型项目 | 3-5 行 | 更细粒度检测 |
| 中型项目 | 5-10 行 | 平衡检测精度与性能 |
| 大型项目 | 10-20 行 | 提高检测效率 |

### 重复代码处理策略
1. **提取函数**：将重复代码提取为独立函数
2. **使用模板方法**：统一相似流程
3. **使用继承**：抽取公共逻辑到父类
4. **使用组合**：通过组合复用代码

### 配置示例
```bash
# 最小 8 行，相似度 85%
python scripts/duplicate_detector.py --path ./myproject --min-lines 8 --min-similarity 0.85
```

## 安全扫描配置

### 严重级别说明
| 级别 | 说明 | 处理优先级 |
|-----|------|-----------|
| High（高危） | 可能导致严重安全漏洞 | 立即修复 |
| Medium（中危） | 存在安全风险 | 尽快修复 |
| Low（低危） | 潜在安全隐患 | 计划修复 |

### bandit 配置文件
可在项目根目录创建 `.bandit` 文件：

```yaml
# 跳过的检查项
skips: ['B101', 'B601']

# 排除的目录
exclude_dirs: ['/tests/', '/migrations/']
```

### 常见安全问题
| 测试ID | 问题类型 | 修复建议 |
|-------|---------|---------|
| B101 | 使用 assert 语句 | 使用 unittest 断言 |
| B105 | 硬编码密码 | 使用环境变量或配置文件 |
| B107 | 硬编码临时文件路径 | 使用 tempfile 模块 |
| B301 | 使用 pickle | 使用 JSON 等安全格式 |
| B601 | 参数化查询 | 使用参数化 SQL |
| B602 | subprocess shell=True | 使用 shell=False |

### 配置示例
```bash
# 只显示高危问题
python scripts/security_scanner.py --path ./myproject --severity high
```

## 最佳实践

### 分析流程建议
1. **开发阶段**：使用宽松配置快速发现问题
2. **测试阶段**：使用严格配置确保质量
3. **发布阶段**：进行全面安全扫描

### 团队协作建议
1. 将配置文件纳入版本控制
2. 在 CI/CD 中集成分析脚本
3. 定期进行全量分析
4. 建立问题修复优先级规范

### 性能优化建议
1. 大型项目使用增量分析
2. 排除不需要分析的目录（venv、node_modules 等）
3. 设置合理的阈值减少误报
4. 使用缓存提高分析速度
